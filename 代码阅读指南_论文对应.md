```python
# 完整的约束损失计算
def planning_loss_with_constraints(
    ego_fut_preds,    # 预测的自车轨迹 [bs, fut_ts, 2]
    ego_fut_gt,       # GT 轨迹
    agent_boxes,      # 他车3D框
    agent_trajs,      # 他车预测轨迹
    map_vectors,      # 矢量化地图
    map_scores        # 地图置信度
):
    # 1. 模仿学习损失
    loss_imitation = F.l1_loss(ego_fut_preds, ego_fut_gt)
    
    # 2. 碰撞约束 (论文 3.4.1)
    loss_collision = compute_collision_loss(
        ego_fut_preds, agent_trajs,
        delta_x=1.5,  # 横向安全距离
        delta_y=3.0   # 纵向安全距离
    )
    
    # 3. 边界约束 (论文 3.4.2)
    loss_boundary = compute_boundary_loss(
        ego_fut_preds, 
        map_vectors[map_type == 'boundary'],
        delta_bd=1.0  # 边界安全距离
    )
    
    # 4. 方向约束 (论文 3.4.3)
    loss_direction = compute_direction_loss(
        ego_fut_preds,
        map_vectors[map_type == 'lane'],
        rho_dir=2.0   # 车道查找半径
    )
    
    return loss_total
```

**关键实现细节：**

```python
# 碰撞约束实现 (约100行)
def compute_collision_loss(ego_traj, agent_trajs):
    for t in range(fut_ts):
        ego_pos = ego_traj[:, t, :]  # [bs, 2]
        agent_pos = agent_trajs[:, :, t, :]  # [bs, num_agent, 2]
        
        # 计算距离
        dist = torch.norm(ego_pos.unsqueeze(1) - agent_pos, dim=-1)
        
        # 找最近的 agent
        min_dist, _ = dist.min(dim=1)
        
        # 计算横向/纵向距离
        dx, dy = compute_lateral_longitudinal(ego_pos, agent_pos)
        
        # 应用阈值
        loss_x = torch.clamp(1.5 - dx, min=0)
        loss_y = torch.clamp(3.0 - dy, min=0)
        
    return (loss_x + loss_y).mean()
```

**阅读建议：**
1. 这是 VAD 的核心创新，一定要理解
2. 对比规则规划中的硬约束 vs 这里的软约束（loss）
3. 理解为什么向量化表示能方便地计算这些约束

## 🔍 VADv2 的关键改进

### 论文改进：从回归到分类

**v1**: 直接回归一条轨迹 → 单模态，难以融合规则
**v2**: 在4096条预定义轨迹上分类 → 多模态，可后处理

### 对应代码位置

**主文件：`VADv2/VADv2_head.py`**

```python
# 1. 轨迹词表 (约100行)
self.trajectory_vocab  # [4096, fut_ts=6, 2]
# 预先定义的4096条候选轨迹，覆盖常见驾驶场景

# 2. 规划头改为分类器 (约1500行)
self.planning_cls_head = nn.Linear(hidden, 4096)
# 输出: [bs, 4096] 的概率分布

# 3. 推理时选择 (约1800行)
def inference():
    logits = self.planning_cls_head(ego_feats)  # [bs, 4096]
    probs = F.softmax(logits, dim=-1)
    
    # 方法1: 选最高概率
    top_idx = probs.argmax(dim=-1)
    pred_traj = self.trajectory_vocab[top_idx]
    
    # 方法2: Top-K 采样，结合规则后处理
    top_k_idx = probs.topk(k=10).indices
    candidates = self.trajectory_vocab[top_k_idx]
    # 可以在这里加规则筛选！
    
    return pred_traj
```

```python
    y_range=(-50, 50),
    # 采样策略: 均匀采样 + 高斯采样 + 专家轨迹聚类
)
```

## 💡 面试必问的代码细节

### Q1: 向量化地图是怎么表示的？
**答：** 每个地图元素用20个有序点 `(x, y)` 表示，代码在 `VAD_head.py` 的 `map_pts_regressor`，输出形状 `[bs, 100, 20, 2]`，意思是最多100个地图元素，每个20个点。

### Q2: 多模态轨迹预测怎么训练的？
**答：** 使用 Min-of-N loss，代码在 `VAD_head.py` 约1100行，只对6个模态中最接近GT的那个计算损失，鼓励多样性。

### Q3: Ego-Agent Interaction 具体怎么做的？
**答：** 用 Transformer Decoder 的 Cross-Attention，ego_query 作为 query，agent_queries 作为 key/value，代码在 `VAD_head.py` 的 `planning_agent_decoder`，约1400行。

### Q4: 三个约束损失怎么平衡的？
**答：** 通过损失权重 `ω_3, ω_4, ω_5`，在配置文件 `VAD_base_e2e.py` 中设置，通常碰撞约束权重最大。

### Q5: VADv2 的4096条轨迹怎么来的？
**答：** 预先生成的词表，代码在 `VADv2_config_voca4096.py`，通过均匀采样驾驶空间 + 聚类专家轨迹得到，覆盖常见场景。
