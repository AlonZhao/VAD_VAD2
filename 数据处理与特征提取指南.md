### Step 4: 理解特征提取（1小时）⭐

**目标**：知道图像如何变成BEV特征

**阅读顺序**：
1. `VAD.py` 的 `extract_feat()` 函数
2. `VAD.py` 的 `extract_img_feat()` 函数
3. `VAD_head.py` 的 `get_bev_features()` 函数

**代码流程**：

```python
# 1. 图像 → 多尺度特征 (VAD.py:extract_feat)
def extract_feat(self, img, img_metas):
    # img: [B, 6, 3, H, W]
    B, N, C, H, W = img.shape  # N=6相机
    img = img.view(B * N, C, H, W)  # 展平batch和相机维
    
    # ResNet backbone
    x = self.img_backbone(img)  
    # 输出4个尺度: [B*6, C_i, H/s_i, W/s_i]
    # s_i = 4, 8, 16, 32
    
    # FPN neck
    x = self.img_neck(x)
    # 输出: list of [B*6, 256, H/s_i, W/s_i]
    
    # 恢复相机维
    img_feats = []
    for feat in x:
        BN, C, H, W = feat.shape
        feat = feat.view(B, N, C, H, W)
        img_feats.append(feat)
    
    return img_feats  # list of [B, 6, 256, H/s, W/s]

# 2. 多尺度特征 → BEV特征 (VAD_head.py:get_bev_features)
def get_bev_features(self, mlvl_feats, img_metas, prev_bev):
    # mlvl_feats: 多尺度图像特征
    
    # 初始化BEV queries
    bs, num_cam, _, _, _ = mlvl_feats[0].shape
    bev_queries = self.bev_embedding.weight  # [H*W, C]
    bev_queries = bev_queries.unsqueeze(1).repeat(1, bs, 1)  # [H*W, B, C]
    
    # BEV Encoder（核心）
    bev_embed = self.transformer.get_bev_features(
        mlvl_feats,      # 图像特征
        bev_queries,     # BEV queries
        self.bev_h,      # BEV高度网格数
        self.bev_w,      # BEV宽度网格数
        grid_length=(self.real_h / self.bev_h, self.real_w / self.bev_w),
        bev_pos=bev_pos,              # BEV位置编码
        img_metas=img_metas,          # 相机参数
        prev_bev=prev_bev,            # 历史BEV
    )
    # 输出: [H*W, B, C] BEV特征
    
    return bev_embed
```

**BEV Encoder内部**（VAD_transformer.py）：
```python
class BEVFormerEncoder(nn.Module):
    def forward(self, bev_query, key, value, *args, **kwargs):
        # bev_query: [H*W, B, C]
        # key, value: 图像特征
        
        for layer in self.layers:
            # 1. Temporal Self-Attention（融合历史BEV）
            if prev_bev is not None:
                bev_query = layer.temporal_self_attn(
                    query=bev_query,
                    key=prev_bev,
                    value=prev_bev,
                )
            
            # 2. Spatial Cross-Attention（从图像采样）
            bev_query = layer.spatial_cross_attn(
                query=bev_query,
                key=key,    # 图像特征
                value=value,
                reference_points=ref_3d,  # BEV→图像投影点
            )
        
        return bev_query  # [H*W, B, C]
```

## 🎯 核心知识点总结

### 数据处理的3个关键点

**1. 预处理：批量计算GT（离线）**
```
nuScenes原始 → 预处理脚本 → .pkl文件
          （运行一次，几小时）
```
- 计算所有ego相关数据（免费午餐）
- 追踪生成agent轨迹
- 保存为中间格式

**2. Dataset：按需读取（训练时）**
```
.pkl文件 → Dataset.__getitem__() → 一个样本
        （每个epoch都读取）
```
- 读取图像（根据路径加载）
- 读取GT（从.pkl读取）
- 应用数据增强

**3. 特征提取：在线计算（前向传播）**
```
图像 → ResNet → FPN → BEV Encoder → BEV特征
  （每个batch都计算）
```
- ResNet提取多尺度特征
- BEV Encoder聚合到鸟瞰图
- 时序融合历史信息

### 面试必答的3个问题

**Q1: 训练时"未来"轨迹从哪来？**
> "在预处理阶段，从ego pose日志往后查询6帧（3秒）得到。因为是离线处理，训练时'未来'已经发生了，所以是事后回看。这就是数据闭环的'免费午餐'——部署时实时采集pose，事后就能自动生成训练GT。代码在 vad_nuscenes_converter.py 的431-459行。"

**Q2: BEV特征是怎么从图像得到的？**
> "通过BEV Encoder的两步attention：1) Spatial Cross-Attention，将BEV queries投影到6个相机平面，用Deformable Attention从图像特征采样；2) Temporal Self-Attention，融合上一帧的BEV特征。这样既聚合了360°视野，又捕获了时序信息。"

**Q3: 哪些数据需要人工标注？**
> "只有3D检测框需要人工标注（类别2），成本高。ego相关数据（历史/未来轨迹、指令、低频特征）完全不需要标注，从日志计算即可。地图从HD Map采样，agent轨迹用追踪算法生成。所以数据闭环的核心瓶颈是3D检测的自动标注。"
