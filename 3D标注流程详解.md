# nuScenes 3D标注流程详解

> 解答：如何在2D图像上标注3D框？VAD如何使用这些标注？

---

## ⚠️ 重要澄清

**VAD是纯视觉方法，不使用点云！**

- ✅ VAD读取：图像 + 已标注好的3D框（作为GT）
- ❌ VAD不读取：点云数据
- 📌 点云只在标注阶段使用（nuScenes团队完成）
- 📌 VAD只读取标注结果，不碰点云文件

详见：`VAD数据使用说明.md`

---

## 🎯 核心问题

**问题**：图像是2D平面，3D检测框是立体的，怎么标注？

**答案**：nuScenes不是在图像上直接标注3D框，而是**在3D点云上标注，然后投影到图像验证**。

## 📊 nuScenes 的标注流程（真实工作流）

```
┌─────────────────────────────────────────────────────────┐
│ 步骤1：采集多模态数据（自动驾驶车辆行驶）                  │
├─────────────────────────────────────────────────────────┤
│ 同步采集：                                               │
│  • LiDAR点云（32线激光雷达，360°，10Hz）                  │
│  • 6个相机图像（前/后/左前/左后/右前/右后，12Hz）          │
│  • Radar数据（5个毫米波雷达）                             │
│  • GPS/IMU（ego位姿）                                    │
│  • CAN总线（速度、转向等）                                │
│                                                          │
│ 关键：所有传感器都**时间同步 + 空间标定**                  │
└────────────────────────┬─────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│ 步骤2：在3D点云上标注（标注员的主要工作）                  │
├─────────────────────────────────────────────────────────┤
│ 标注工具：专用的3D标注软件（类似CVAT 3D）                  │
│                                                          │
│ 操作流程：                                                │
│  1. 加载LiDAR点云（显示为3D点云）                         │
│  2. 标注员在3D视图中框选物体                              │
│     - 旋转视角观察物体（鸟瞰图、侧视图、透视图）            │
│     - 拖动鼠标绘制3D立方体包围框                          │
│     - 调整框的8个顶点，使其紧贴物体                        │
│  3. 标注类别（car, truck, pedestrian等）                  │
│  4. 标注朝向（物体的前进方向）                             │
│  5. （可选）标注属性（moving/parked, with_rider等）        │
│                                                          │
│ 输出：3D框的7个参数                                       │
│   • center: (x, y, z) - 中心坐标（LiDAR坐标系）           │
│   • size: (w, l, h) - 宽度、长度、高度                    │
│   • rotation: yaw角 - 朝向（绕z轴旋转）                   │
│   • category: 类别                                       │
│   • instance_token: 唯一ID（用于跨帧追踪）                │
└────────────────────────┬─────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│ 步骤3：投影到图像验证（质量检查）                          │
├─────────────────────────────────────────────────────────┤
│ 为什么要投影？                                            │
│   点云上的标注可能不准（遮挡、稀疏），需要图像辅助验证      │
│                                                          │
│ 投影流程：                                                │
│  1. 读取3D框的8个角点坐标（LiDAR系）                      │
│  2. 使用标定矩阵投影到6个相机平面                          │
│     3D点 → 2D点：p_img = K @ [R|t] @ p_lidar             │
│     其中：                                                │
│       K: 相机内参矩阵（焦距、光心）                        │
│       [R|t]: LiDAR→相机的外参（旋转+平移）                │
│  3. 在图像上绘制3D框的投影（12条边的线段）                 │
│  4. 标注员检查：框是否准确包围物体？                       │
│     - 如果不准，回到步骤2调整3D框                          │
│     - 如果准确，标注完成                                  │
│                                                          │
│ 可视化效果：                                              │
│   你在网上看到的"图像上的3D框"就是这样来的！               │
│   它不是直接在图像上画的，而是3D框投影过来的。             │
└────────────────────────┬─────────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────────┐
│ 步骤4：跨帧追踪（建立时序关联）                            │
├─────────────────────────────────────────────────────────┤
│ 目标：同一辆车在不同帧中保持同一个 instance_token          │
│                                                          │
│ 方法：                                                    │
│  1. 标注员手动关联（对于关键帧）                           │
│     - 在t帧标注一个车后，跳到t+1帧                         │
│     - 找到同一辆车，标注为"same instance"                 │
│  2. 算法辅助插值（对于中间帧）                             │
│     - 在关键帧之间线性插值3D框                             │
│     - 自动生成中间帧的标注                                 │
│  3. 人工修正（质量保证）                                   │
│     - 检查插值是否准确                                     │
│     - 修正错误的追踪                                       │
└─────────────────────────────────────────────────────────┘
```

## 🔍 为什么在3D点云上标注而不是图像？

### 对比：2D标注 vs 3D标注

| 方法 | 优点 | 缺点 |
|------|------|------|
| **在图像上直接标3D框** | 直观 | ❌ 深度信息不准（透视变形）<br>❌ 需要6个相机都标注<br>❌ 不同视角的框难以对齐 |
| **在点云上标3D框** ✅ | ✅ 深度信息准确（LiDAR测距）<br>✅ 一次标注，多视角共享<br>✅ 物理尺寸准确 | 需要点云数据（成本高） |

### 实际例子

**场景**：标注一辆距离20米的车

**方法1：在图像上直接画3D框**
```
问题：
- 如何确定车的深度？（是19m还是21m？）
- 如何确定车的尺寸？（长4m还是5m？）
- 如何确定朝向？（车头向左30°还是35°？）
- 6个相机都要画，而且必须一致
```

**方法2：在点云上标注 ✅**
```
优势：
- LiDAR直接测距：深度=20.3m（精度±5cm）
- 点云显示物体形状：长4.2m × 宽1.8m × 高1.5m
- 点云显示朝向：车头明确指向某个方向
- 标一次，自动投影到6个相机验证
```

## 🖼️ 你看到的可视化是怎么来的？

### 网上看到的"图像上的3D框"

```python
# nuScenes 可视化代码（简化版）
def render_annotation(sample_data, annotation):
    """在图像上绘制3D框的投影"""
    
    # 1. 读取3D框（在LiDAR坐标系）
    box_3d = Box(
        center=[10.5, 2.3, 0.5],    # 中心 (x,y,z)
        size=[4.2, 1.8, 1.5],       # 尺寸 (l,w,h)
        orientation=Quaternion(axis=[0,0,1], angle=0.3)  # 朝向
    )
    
    # 2. 获取8个角点（3D空间）
    corners_3d = box_3d.corners()  # [3, 8] 8个角点的xyz坐标
    #     5------4
    #    /|     /|
    #   1------0 |
    #   | 6----|-7
    #   |/     |/
    #   2------3
    
    # 3. 投影到图像（LiDAR → 相机）
    # 3.1 坐标变换：LiDAR → ego → 相机
    corners_3d = transform(corners_3d, lidar2ego @ ego2cam)
    
    # 3.2 透视投影：3D → 2D
    corners_2d = cam_intrinsic @ corners_3d  # [3, 8]
    corners_2d = corners_2d[:2] / corners_2d[2]  # 归一化，得到像素坐标 [2, 8]
    
    # 4. 在图像上绘制12条边
    # 前面4条边
    draw_line(img, corners_2d[0], corners_2d[1])  # 0-1
    draw_line(img, corners_2d[1], corners_2d[2])  # 1-2
    draw_line(img, corners_2d[2], corners_2d[3])  # 2-3
    draw_line(img, corners_2d[3], corners_2d[0])  # 3-0
    # 后面4条边
    draw_line(img, corners_2d[4], corners_2d[5])  # 4-5
    ...
    # 连接前后4条边
    draw_line(img, corners_2d[0], corners_2d[4])  # 0-4
    ...
    
    return img
```

**关键理解**：
- 你看到的3D框不是"画"在图像上的
- 而是先在3D空间标注，然后**投影**到图像平面
- 12条边的线段是投影后的结果

## 💰 nuScenes 标注成本

### 标注数据量

```
nuScenes 数据集统计：
- 1000个场景，每个约20秒
- 40,000个关键帧（2Hz采样）
- 1.4M 个3D框标注
- 平均每帧35个3D框
```

### 标注时间估算

```
单帧标注时间：
- 新场景第一帧：10-15分钟（标注所有物体 + 分配instance_token）
- 追踪帧：2-3分钟（调整框位置 + 验证追踪）
- 质量检查：1-2分钟

平均每帧：约5分钟

总标注时间：
40,000帧 × 5分钟 = 200,000分钟 ≈ 3,333小时

如果10个标注员并行：333小时 ≈ 42个工作日（8小时/天）
```

### 标注成本

```
人力成本：
- 专业标注员：$20-30/小时（发达国家）
- 或 $5-10/小时（外包到发展中国家）

总成本估算：
- 高端：3,333小时 × $30 = $100,000
- 外包：3,333小时 × $10 = $33,000

加上：
- 标注工具开发：$50,000-100,000
- 质量审核：+30%成本
- 数据采集（车辆、设备、司机）：$500,000+

nuScenes 总成本：$1-2M（百万美元级）
```

**这就是为什么数据闭环要追求"自动标注"——人工成本太高了！**

## 🤖 自动标注的方向（数据闭环的关键）

### 方法1：大模型自动标注（当前主流）

```
实车采集数据（LiDAR + 相机）
    ↓
离线大模型检测（参数量远大于车端模型）
    ↓
后处理 + 人工抽检（10%）
    ↓
生成训练GT
```

**优势**：
- 成本降低90%（人工只需抽检）
- 速度快（GPU并行处理）
- 一致性好（模型不会疲劳）

**案例**：
- Tesla：Occupancy Network自动标注
- 华为：盘古大模型自动标注
- Waymo：多模态融合自动标注

---

### 方法2：弱监督标注

```
只标注关键帧（每秒1帧）
    ↓
追踪算法自动插值中间帧
    ↓
人工修正错误追踪（10%）
```

**优势**：
- 标注量减少50%
- 利用物体运动连续性

---

### 方法3：主动学习

```
模型预测置信度
    ↓
只标注低置信度样本（困难样本）
    ↓
模型重训练
```

**优势**：
- 聚焦困难场景
- 标注效率高

## 🎯 为什么VAD需要3D标注？

### 模型需要的监督信号

```python
# VAD的检测损失
loss_det = L_cls(pred_cls, gt_labels_3d) + L_reg(pred_boxes, gt_bboxes_3d)
                                                              ↑
                                                  需要精确的3D坐标
```

**如果只有2D标注**：
- ❌ 无法监督深度预测
- ❌ 无法监督物体尺寸
- ❌ 无法监督3D朝向
- ❌ 规划约束无法计算（碰撞距离需要3D位置）

**有了3D标注**：
- ✅ 端到端学习深度感知
- ✅ 学习物理尺寸估计
- ✅ 学习物体朝向
- ✅ 支持3D规划约束

## 🎓 面试加分点

**当被问到"3D标注怎么做"**：
> "nuScenes是在LiDAR点云上标注3D框，而不是图像上。因为点云提供了准确的深度和尺寸信息，标注员在3D视图中框选物体，然后投影到6个相机图像验证准确性。这种方法确保了3D坐标的精度，但成本很高——nuScenes花费了约3,333小时人工。这就是为什么数据闭环需要大模型自动标注，把成本降低90%。"

**展示工程理解**：
> "网上看到的'图像上的3D框'其实是投影的结果，用的是相机内外参矩阵：`corners_2d = K @ [R|t] @ corners_3d`。这12条边是8个3D角点投影到2D平面连接起来的。VAD训练时需要这些3D标注来监督深度预测和规划约束，但部署时模型自己生成3D框，不需要标注。"
