# VAD / VADv2 一周学习计划

> **背景**：从纯规则决策规划转向端到端规划，目标是面试相关岗位。
> **约束**：一周时间、无 GPU、不实际训练，纯阅读 + 理解 + 面试准备。
> **核心目标**：一周后能在面试中清晰讲出 VAD 是什么、怎么做的、关键代码怎么实现、相比规则规划的优劣。

**`VAD_transformer.py`（488 行，2h）**

### Day 4（周四）：规划模块（最重要）— 4 小时

**`VAD_head.py` 规划相关部分 + `VADv2/VADv2_head.py`（4h）**

这是**主战场**，作为规划工程师，面试官会重点问这里。

**v1 的规划：**
- planning query 与 map query、agent query 做 cross-attention
- 直接回归一条 6×2 的未来轨迹
- 加了 3 个约束损失：碰撞、车道边界、方向

**v2 的规划：**
- 预定义 4096 条候选轨迹（vocabulary）
- 模型输出每条的概率分布
- 多模态、可枚举、便于加规则后处理

**作为规则工程师，重点对比思考：**

| 维度       | 规则规划 | VAD v1     | VAD v2         |
| ---------- | -------- | ---------- | -------------- |
| 多模态     | 显式枚举 | 单模态     | 词表分类       |
| 安全约束   | 硬约束   | loss 软约束 | 可后处理筛选   |
| 可解释性   | 强       | 弱         | 中（可看概率） |

## 三、关键省时建议

1. **不要钻细节**：mmcv/mmdet3d 框架代码不用读，看 VAD 自己的代码就行
2. **不要纠结环境**：没显卡就别折腾，安装失败会打击信心
3. **善用论文图**：VAD 论文 Figure 2 架构图比读代码效率高 10 倍
4. **录音讲解**：每天对着代码自己讲一遍并录音，回放发现没讲清楚的地方
5. **一句话原则**：每个模块都能用一句话讲清楚，讲不清就是没懂

## 四、"作弊小抄"

**VAD 一句话总结：**
> 用向量化的 query 表示地图、他车、自车规划，通过 Transformer 交互一次性输出未来轨迹，比稠密栅格方案更快、更易加约束。

**VAD v2 一句话总结：**
> 把回归改成在 4096 条预定义轨迹上分类，输出概率分布，多模态友好，便于和规则后处理结合。
