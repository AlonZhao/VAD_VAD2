## 1. 技术演进时间线

```
2020   Deformable DETR (ECCV)
       └─ 稀疏采样注意力，解决 Transformer 计算瓶颈
         │
2021   DETR3D (CoRL)
       └─ 3D 检测用 query，参考点迭代 refine
         │
2022   BEVFormer (ECCV) ★ 里程碑
       └─ 时空 BEV 表示 + Spatial/Temporal Attention
         ├──────────────┬──────────────┬──────────────┐
         ↓              ↓              ↓              ↓
       MapTR          PETR         StreamPETR    PETRv2
     (向量化地图)   (位置编码)    (时序流式)   (时序改进)
         │              │              │              │
         └──────────────┴──────┬───────┴──────────────┘
                               ↓
2023   UniAD (CVPR) ★ 端到端元年
       └─ 统一框架：检测→跟踪→地图→运动→规划，稠密栅格规划
         │
       VAD (ICCV) ★ 向量化端到端
       └─ 向量化场景表示 + ego query 交互 + 轻量快速
         │
2024   VADv2 (arXiv)
       └─ 概率规划（轨迹词表分类）+ 多维安全评分
         │
       ┌─────────────┼─────────────┐
       ↓             ↓             ↓
    Senna       DiffusionDrive    RAD
   (激进优化)  (扩散模型规划)  (RL后训练)
```

**关键转折点**：
- **BEVFormer**：统一了多视角感知的表示方法（BEV）
- **UniAD**：证明了端到端全链路在一个模型里可行
- **VAD**：提出向量化 + ego query 交互范式，更轻量
- **VADv2**：从回归改分类，引入可控性

## 2. 核心技术对比矩阵

| 方案 | 场景表示 | 规划方式 | 速度(FPS) | L2(m) | 核心创新 |
|---|---|---|---|---|---|
| **BEVFormer** | 稠密BEV | 无规划 | ~10 | — | 时空BEV奠基 |
| **UniAD** | 稠密BEV | 栅格cost map | ~2 | 0.7 | 端到端全链路统一 |
| **VAD** | 向量化query | ego交互回归 | ~15 | 1.5 | 轻量+快速 |
| **VADv2** | 向量化query | 词表分类 | ~15 | 1.2 | 可控+多模态 |
| **Senna** | 向量化 | 激进优化 | ~20 | 0.9 | 速度极致优化 |
| **DiffusionDrive** | 混合 | 扩散模型 | ~5 | 0.6 | 多模态生成 |
| **RAD** | 向量化 | RL+词表 | ~12 | 1.0 | 闭环RL后训练 |

**解读**：
- UniAD：最全面但慢（稠密表示开销大）
- VAD：速度和精度平衡，工程友好
- DiffusionDrive：精度高但慢，难部署
- RAD：结合 RL 提升闭环性能

## 3. 逐个方案详解

### 3.1 BEVFormer（2022 ECCV）

**贡献**：
- **Spatial Cross-Attention**：BEV query 通过 lidar2img 投影主动去图像采样
- **Temporal Self-Attention**：融合 prev_bev，用 ego motion 对齐

**不足**：
- 只做感知（检测+分割），不做规划
- 稠密 BEV 特征图（100×100×256）内存开销大

**影响**：
- 成为后续端到端方案的"感知主干"（VAD/UniAD 都直接用）

### 3.2 UniAD（2023 CVPR Best Paper Finalist）

**架构**：
```
图像 → BEVFormer → 检测 → 跟踪 → 地图 → 运动预测 → 规划
                    (Det)  (Track) (Map)  (Motion)   (Plan)
                                                      ↑
                                            稠密 cost map 规划
```

**规划方式**：
```python
cost_map = collision_cost + boundary_cost + comfort_cost  # 栅格cost
plan_traj = A* / sampling 在 cost_map 上搜索最优路径
```

**优势**：
- ✅ 全链路端到端，证明了可行性
- ✅ L2 = 0.7m，开环精度 SOTA
- ✅ 显式建模跟踪（ID 关联），运动预测更准

**劣势**：
- ❌ 慢（~2 FPS）——稠密表示 + 5 个任务串联
- ❌ 栅格规划不如向量化灵活（离散化误差）
- ❌ 闭环上抖动（论文 Fig.5 承认）

### 3.3 VAD（2023 ICCV）

**核心差异**：
```
UniAD: 稠密BEV → 稠密 cost map → 栅格搜索
VAD:   稠密BEV → 向量化query → ego交互回归
```

**向量化**：
- 300 个 agent query（不是稠密特征图）
- 100 条 map query（polyline，不是栅格）
- 1 个 ego query 和它们 cross-attention

**优势**：
- ✅ 快（~15 FPS）——query 数量远少于栅格像素
- ✅ 轻量（参数少、内存小）
- ✅ 闭环更稳（论文对比 UniAD，抖动更少）

**劣势**：
- ❌ L2 = 1.5m，略逊 UniAD（向量化表示能力弱于稠密）
- ❌ 跳过了跟踪模块（直接用 detection query）

### 3.4 VADv2（2024 arXiv）

**唯一改动**：规划头从**回归**改**分类**

```
v1: ego_feats → MLP → 3条轨迹 (x,y 直接回归)
v2: ego_feats → 给4096条预定义轨迹打分 → softmax选最高分
```

**轨迹词表**：
- 对训练集真实轨迹聚类（k-means），得到 4096 个代表性"开法"
- 模型只学"选哪条"，不学"生成坐标"

**多维打分**：
```python
col_score    # 会不会撞车
bd_score     # 会不会出边界
cl_score     # 偏不偏中心线
expert_score # 像不像人类开的
```

**优势**：
- ✅ 多模态更强（4096 条 vs v1 的 3 条）
- ✅ 可解释（能看每条轨迹的各维度分数）
- ✅ 可后处理（筛掉危险轨迹再选）

**劣势**：
- ❌ 词表覆盖不全的场景会失效（极端场景没见过）
- ❌ 推理时要过 4096 条，比 v1 略慢

### 3.5 Senna（2024）

**定位**：VAD 的"激进优化版"

**优化手段**：
- 更少的 query（agent 200 vs VAD 300）
- 更少的 decoder 层（2 层 vs 3 层）
- 剪枝不重要的注意力头

**结果**：
- 速度 ~20 FPS（比 VAD 还快 33%）
- L2 = 0.9m（精度反而提升）

**启示**：VAD 架构还有冗余，精细调优能做得更好。

### 3.6 DiffusionDrive（2024）

**核心思想**：把规划建模成**条件扩散生成**

```
噪声轨迹 → 去噪网络(条件=场景特征) → 逐步生成清晰轨迹
```

**优势**：
- ✅ L2 = 0.6m（目前最低）
- ✅ 多模态天然支持（扩散过程本身就是概率分布）
- ✅ 生成质量高（轨迹平滑）

**劣势**：
- ❌ 慢（~5 FPS）——去噪迭代 50+ 步
- ❌ 难部署（扩散模型工程复杂）
- ❌ 不可控（生成过程黑盒，难加约束）

### 3.7 RAD（Reinforcement Learning Augmented）（2024）

**核心思想**：在 VADv2 基础上加 **RL 后训练**

```
预训练: 模仿学习（专家轨迹监督）
        ↓
后训练: 在 CARLA 闭环里用 RL 微调
        reward = -碰撞 - 偏离 + 路径完成
```

**优势**：
- ✅ 闭环性能提升明显（碰撞率降 30%）
- ✅ 学会"预判"其他车反应（RL 在交互中学到的）
- ✅ 词表提供离散动作空间，RL 易优化

**劣势**：
- ❌ 训练成本高（需要大量仿真交互）
- ❌ reward 设计难（手调权重）
- ❌ sim-to-real gap（仿真学到的不一定真车适用）

## 4. 面试常见对比问题

### Q1: VAD 和 UniAD 的核心区别？

> **场景表示**：UniAD 用稠密 BEV 栅格，VAD 用向量化 query。
>
> **规划方式**：UniAD 在 cost map 上搜索路径，VAD 用 ego query 和环境交互后直接回归。
>
> **速度 vs 精度**：UniAD 慢但精度高（L2=0.7m），VAD 快但略低（1.5m）。工程上 VAD 更实用——15 FPS 能满足实时性，1.5m 误差在车道内也够用。
>
> **我的选择**：从规则规划转端到端，我会选 VAD。因为规则规划背景更习惯"显式表示"（向量化车道、agent），VAD 的 query 范式更接近我原来的思维；而且速度快意味着迭代快，更适合快速验证想法。

### Q2: 为什么 VADv2 要从回归改分类？

> **多模态问题**：回归是单峰的——遇到"左转右转都行"的歧义场景，网络容易学成两者平均（开到中间撞上）。分类天然支持多峰：左转一个概率峰、右转一个峰，不会平均。
>
> **可控性**：v1 的安全约束是训练时的 loss，推理时看不见。v2 把约束变成每条轨迹的打分（col/bd/cl），推理时可见可控——我能在选轨迹前用规则硬筛掉所有危险的，再从安全里选最优。这对规则规划背景是很自然的衔接。
>
> **局限**：词表覆盖不全的极端场景会失效。但 4096 条已经很全了（聚类训练集），实际上够用。

### Q3: 扩散模型规划（DiffusionDrive）为什么没火？

> **速度瓶颈**：去噪迭代 50 步，5 FPS 达不到实时（10Hz 控制器要求）。
>
> **工程复杂**：扩散模型部署难优化（TensorRT 支持不好），车规芯片跑不动。
>
> **不可控**：生成过程是黑盒，想加"别走这个区域"的硬约束很难——不像 VADv2 的词表，我能直接筛掉不合规的。
>
> **适用场景**：实验室刷榜可以（精度高），但量产落地选 VAD 类方案（快+可控）。扩散更适合离线轨迹优化，不适合在线规划。

### Q4: RL 后训练（RAD）的价值在哪？

> **闭环 gap**：纯模仿学习在开环数据上训，闭环交互场景泛化差。RL 直接在闭环里学，能学到"其他车会对我反应"这种交互策略。
>
> **但成本高**：需要稳定的仿真环境、大量交互样本、调 reward 权重。对小团队不友好。
>
> **我的看法**：RL 是"锦上添花"，不是必需。VAD 基础版已经够用，有余力再加 RL。而且 RL 的 sim-to-real gap 也大——仿真里学到的激进策略，真车不一定敢用（安全冗余不够）。我会优先保证模仿学习基座扎实，再考虑 RL。

### Q5: 下一代端到端会往哪个方向走？

个人判断三个方向：

1. **世界模型 + 规划**：用 Transformer 预测"如果我这么开，世界会变成什么样"，在想象中做规划（类似 AlphaGo 搜索树）。Wayve 的 GAIA-1 在做这个。

2. **大模型加持**：用 VLM（视觉语言模型）理解场景语义（"前方施工，绕行"），结合端到端执行。Tesla FSD v12 疑似在做。

3. **轻量化 + 车规**：现在模型太重（BEVFormer 几百MB），往车规芯片（地平线J5、Orin）部署难。下一代要"小而强"——Senna 证明了剪枝有效，量化、蒸馏、NAS 都会跟上。

## 5. 技术选型决策树（实战）

```
需求: 要做端到端规划
    │
    ├─ 追求开环精度（刷榜）？
    │   └─ Yes → DiffusionDrive (L2最低)
    │   └─ No ↓
    │
    ├─ 有大算力卡（A100）+ 不急部署？
    │   └─ Yes → UniAD (全链路最全)
    │   └─ No ↓
    │
    ├─ 要快速迭代 + 实时性？
    │   └─ Yes → VAD / Senna (15-20 FPS)
    │   └─ No ↓
    │
    ├─ 需要可控性 + 规则兜底？
    │   └─ Yes → VADv2 (词表+打分)
    │   └─ No ↓
    │
    ├─ 有仿真环境 + 资源做RL？
    │   └─ Yes → RAD (闭环优化)
    │   └─ No → VAD v1 (基础版足够)
```

**我的选择**（规则规划背景）：
- 起步：**VAD** v1（快速验证端到端可行性）
- 优化：**VADv2**（加规则后处理，保安全）
- 长期：观望世界模型方向，短期不碰 RL（成本高）

## 6. 小结：一张表记住核心差异

| 维度 | UniAD | VAD | VADv2 | 我的评价 |
|---|---|---|---|---|
| **表示** | 稠密 | 向量化 | 向量化 | 向量化更工程友好 |
| **速度** | 慢(2) | 快(15) | 快(15) | 速度是量产硬指标 |
| **精度** | 高(0.7) | 中(1.5) | 中(1.2) | 1.5m 够用，不是瓶颈 |
| **可控** | 弱 | 弱 | 强 | v2 的打分机制是亮点 |
| **闭环** | 抖 | 稳 | 稳 | 开环高≠闭环好 |

**三句话**：
1. BEVFormer 奠定感知基座，UniAD 证明端到端可行，VAD 向量化轻量化。
2. VADv2 回归改分类是关键改进，可控性对规则转端到端很友好。
3. 下一代看世界模型 + 轻量化，RL 是锦上添花不是必需。
