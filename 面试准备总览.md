## 💼 针对"规则规划转端到端"背景的特殊准备

### 面试官必问的3个问题

**Q1: 你做规则规划的，为什么要转端到端？**

**标准答案框架：**
```
痛点识别：
- 规则规划的局限：手工特征、难以处理长尾、调参困难
- 端到端的优势：数据驱动、联合优化、泛化性更好

个人动机：
- 看到行业趋势（特斯拉FSD、Waymo的端到端尝试）
- 希望掌握新技术栈
- 认为未来是端到端+规则的混合方案

学习路径：
- 系统学习VAD作为代表性工作
- 理解矢量化表示（与规则规划的lattice相似）
- 思考如何融合两种方法
```

---

**Q2: 端到端能解决规则规划的哪些痛点？哪些做不到？**

**标准答案：**

**能解决的痛点：**

| 规则规划痛点 | 端到端方案 |
|------------|----------|
| 手工特征设计 | 自动学习特征 |
| 长尾场景覆盖难 | 数据驱动泛化 |
| 模块误差累积 | 联合优化 |
| 调参工作量大 | 一次训练 |

**做不到的：**

| 需求 | 规则规划优势 | 端到端劣势 |
|------|------------|----------|
| 可解释性 | 每步决策可追溯 | 黑盒模型 |
| 安全验证 | 形式化证明 | 难以验证 |
| 边界情况控制 | 显式约束 | 软约束可能违反 |
| 冷启动 | 无需数据 | 依赖大量标注 |

**结论：** 混合方案最优（端到端提供候选，规则做安全检查）

---

**Q3: VADv2的轨迹词表和规则规划的lattice有什么相似和不同？**

**标准答案：**

**相似之处：**
- 都是预先定义候选轨迹集合
- 都是"选择"而非"生成"
- 都便于加约束筛选

**不同之处：**

| 维度 | Lattice（规则） | Trajectory Vocab（VADv2） |
|------|---------------|------------------------|
| 生成方式 | 解析式（Frenet坐标） | 数据驱动聚类 |
| 评分方式 | Cost函数（手工设计） | 神经网络分类器 |
| 候选数量 | 几百条（实时生成） | 4096条（预先固定） |
| 覆盖范围 | 完全覆盖 | 依赖训练数据 |
| 可控性 | 完全可控 | 部分可控（词表设计） |

**融合思路：**
```python
# VADv2输出Top-K
topk_trajs = VADv2.predict(images, k=10)

# 规则规划筛选
for traj in topk_trajs:
    if rule_based_safety_check(traj):
        return traj

# 如果都不安全，回退到规则生成
return lattice_planner.emergency_trajectory()
```

## 🎤 面试实战技巧

### 回答问题的STAR结构

**S (Situation)**: 问题背景
**T (Task)**: 需要解决什么
**A (Action)**: VAD的方案
**R (Result)**: 效果和局限

**示例：**
```
面试官问："VAD如何解决碰撞风险？"

错误回答：
"VAD用了碰撞约束损失。"（太简短）

正确回答：
【S】传统方法用密集占用图判断碰撞，计算昂贵。
【T】VAD需要高效的碰撞约束方式。
【A】VAD利用矢量化轨迹，直接计算ego与agent的点到点距离，
    横向阈值1.5m，纵向3.0m（符合驾驶习惯）。
【R】训练时作为软约束引导安全轨迹，消融实验显示碰撞率降低29%。
    但仍是软约束，极端情况可能违反，需要规则后处理兜底。
```


### 常见追问及应对

**追问1: "VAD在真实车上部署有什么挑战？"**

回答要点：
- 计算资源：4.5 FPS还不够，需要优化到30+ FPS
- 传感器标定：BEV需要精确的相机内外参
- 长尾场景：施工、异常天气的鲁棒性
- 安全验证：如何证明端到端模型的安全性？
- 数据闭环：如何持续收集corner case并重训练

---

**追问2: "你觉得端到端规划的未来是什么？"**

回答要点（展示行业视野）：
- **短期**：混合方案（端到端+规则）成为主流
- **中期**：VAD → VADv2 → RAD（RL后训练）的演进
- **长期**：结合大模型（如Senna），实现更强的泛化和推理
- **关键挑战**：安全验证、法规合规

参考后续工作：
- VADv2（概率规划）
- RAD（3DGS + RL）
- Senna（VLM增强）
- DiffusionDrive（扩散模型）
