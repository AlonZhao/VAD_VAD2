# VAD 数据使用说明（关键澄清）

> **核心澄清**：VAD是纯视觉方法，**不使用点云**，只使用**相机图像 + 已标注好的3D框**

---

## ✅ 正确理解：VAD读取的数据

### 1. **VAD训练时读取什么？**

```python
# VAD.forward_train() 的输入
forward_train(
    img=...,              # ✅ 6个相机图像（VAD真正使用的）
    points=None,          # ❌ 点云（VAD不用，设为None）
    gt_bboxes_3d=...,     # ✅ 3D框GT（nuScenes已标注好的）
    gt_labels_3d=...,     # ✅ 类别GT（nuScenes已标注好的）
    ...
)
```

**关键点**：
- ✅ VAD只用**图像**做输入（纯视觉）
- ✅ 3D框是**已经标注好的GT**，用来监督训练
- ❌ 点云**不参与训练和推理**，VAD完全不读取点云

## 🎯 关键澄清

### **点云在哪里用？**

**只在标注阶段用！**

```
标注流程（nuScenes团队完成）：
1. 标注员在点云上画3D框    ← 点云在这里用
2. 保存到annotations.json
3. 发布nuScenes数据集

VAD使用流程（你要做的）：
1. 读取annotations.json    ← 直接读取标注结果
2. 读取图像文件
3. 训练模型
4. ❌ 从不读取点云文件
```

**类比理解**：
- 就像ImageNet数据集，图像都已经标注好类别了
- 你训练时只读取"图像+类别标签"，不需要知道标注员怎么标的
- nuScenes的3D框也一样，已经标注好了，你直接读取使用

## 💡 完整的问答澄清

### **Q: 标注是如何标注的？**
**A**: nuScenes团队用LiDAR点云标注（这是标注阶段，在你使用数据集之前就完成了）

### **Q: VAD读取点云吗？**
**A**: ❌ 不读取！VAD只读取：
- ✅ 图像（输入）
- ✅ 已标注好的3D框（GT，用于监督）

### **Q: 那3D框信息怎么来的？**
**A**: nuScenes团队在点云上标注好，保存在 `annotations.json` 中，VAD直接读取这个文件。

### **Q: 为什么叫"纯视觉"方法？**
**A**: 因为VAD推理时只需要相机图像，不需要LiDAR。虽然训练时用的GT是LiDAR标注的，但模型本身只处理图像。

## 🎓 面试时的正确说法

### ❌ 错误说法
> "VAD用点云和图像做输入"

### ✅ 正确说法
> "VAD是纯视觉方法，只用6个相机的图像做输入。训练时使用nuScenes已标注好的3D框作为GT监督。这些3D框虽然是在点云上标注的，但VAD本身不读取点云，只读取标注结果。"

### 💯 加分说法
> "VAD的创新之一就是纯视觉达到了接近多模态（相机+LiDAR）的性能。nuScenes提供了LiDAR标注的高质量3D框GT，但VAD推理时完全不需要LiDAR，只用图像就能预测3D框。这对于降低传感器成本很有价值——LiDAR一套要几万美元，而相机只要几百。"

## 🎯 三个阶段的数据使用

### 阶段1：标注阶段（nuScenes团队，你看不到）
```
原始采集：LiDAR点云 + 6相机图像
    ↓
标注员在点云上标注3D框
    ↓
保存：annotations.json（包含3D框坐标）
    ↓
发布：nuScenes数据集（给全世界使用）
```

### 阶段2：训练阶段（VAD，你要做的）
```
读取：
  - 6相机图像 ✅
  - annotations.json中的3D框 ✅（作为GT）
  - 点云 ❌（不读取）
    ↓
训练：
  图像 → 模型 → 预测3D框
  Loss = 预测3D框 vs GT的3D框
    ↓
输出：训练好的模型权重
```

### 阶段3：推理阶段（部署，实车）
```
输入：
  - 6相机实时图像 ✅
  - 点云 ❌（不需要）
  - GT ❌（不需要，模型自己预测）
    ↓
推理：
  图像 → 模型 → 预测3D框
    ↓
输出：预测的3D框、地图、轨迹
```
